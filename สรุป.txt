1. ระบบแกนหลักของเว็บไซต์ (Core System)
ไฟล์ที่เกี่ยวข้อง: css/core.css, js/core.js, read me.txt
ไฟล์ core.js เป็นไฟล์ที่ถูกเรียกใช้ในทุกๆ หน้าของเว็บไซต์ โดยมีการนำระบบหลายๆ อย่างมารวมไว้ในไฟล์เดียวเพื่อให้ทำงานได้อย่างมีประสิทธิภาพ

หน้าที่และการทำงานของโค้ด (core.js):

ระบบ Authentication (เช็คการล็อกอิน): โค้ดจะตรวจสอบว่ามี loggedInUser อยู่ใน localStorage หรือไม่ หากไม่มีและผู้ใช้ไม่ได้อยู่ในหน้า Login โค้ดจะเตะผู้ใช้กลับไปหน้า Login ทันที

ระบบ Navbar & Dropdown: ดึงข้อมูลผู้ใช้ปัจจุบันมาแสดงบนแถบเมนูด้านบน (จำนวนเหรียญ, รูปโปรไฟล์) และสร้างเมนู Dropdown สำหรับเมนู Profile, Shop, Quests, และ Logout

ระบบ Daily Reset (รีเซ็ตรายวัน): ตรวจสอบวันที่ปัจจุบัน (new Date().toDateString()) หากไม่ตรงกับวันที่บันทึกไว้ล่าสุด จะทำการรีเซ็ตภารกิจรายวัน (Daily Quests) ของผู้ใช้คนนั้นให้ใหม่

ระบบ Theme Switcher (เปลี่ยนธีมสี): ดึงข้อมูล "ธีมที่ปลดล็อกแล้ว" ของผู้ใช้ หากผู้ใช้มีสิทธิ์ โค้ดจะเปลี่ยน class ของ <body> เป็นธีมนั้นๆ (เช่น dark, halloween) และบันทึกค่าลง localStorage

ระบบ Global Achievement Toast: เป็นฟังก์ชัน showAchievementToast(achId) ที่สามารถถูกเรียกจากหน้าไหนก็ได้ เพื่อโชว์ Popup เด้งขึ้นมามุมขวาล่างเมื่อผู้เล่นปลดล็อกความสำเร็จใหม่

2. ระบบเข้าสู่ระบบและสมัครสมาชิก (Login & Registration)
ไฟล์ที่เกี่ยวข้อง: Login.html, js/login.js, css/login.css

หน้าที่และการทำงานของโค้ด (login.js):

สลับโหมด Login / Register: ควบคุมการแสดงผลฟอร์ม หากเป็นหน้าสมัครสมาชิกจะแสดงช่องกรอก Email เพิ่มขึ้นมา

การสมัครสมาชิก (Register): * รับค่า Username, Email, Password

การตรวจสอบความถูกต้อง (Validation): บังคับว่า Email ต้องลงท้ายด้วย @gmail.com เท่านั้น, Username ห้ามมีสัญลักษณ์ @ หรือคำว่า gmail, และ Password ห้ามใช้ภาษาไทย

หากผ่านเงื่อนไข จะสร้าง Object ของผู้ใช้ใหม่ (เก็บข้อมูล score, coins, achievements, quests เป็นค่าเริ่มต้น) และ push ลง Array ใน localStorage

การเข้าสู่ระบบ (Login): ค้นหา Username และ Password ในระบบ หากถูกต้องจะเซ็ตค่า loggedInUser และเช็คว่าผู้ใช้นี้เคยดูหน้า Tutorial หรือยัง (hasSeenTutorial) ถ้ายังจะพาไปหน้า Tutorial ถ้าเคยแล้วจะพาไปหน้า Home

3. หน้าหลักและหน้าแนะนำการใช้งาน (Home & Tutorial)
ไฟล์ที่เกี่ยวข้อง: home.html, js/home.js, tutorial.html, js/tutorial.js

หน้าที่และการทำงานของโค้ด:

home.js: จัดการ Popup "Welcome [ชื่อผู้ใช้]" แบบเต็มหน้าจอที่จะแสดงขึ้นมาเพียงครั้งเดียวต่อการเข้าใช้งาน (ใช้ sessionStorage ในการจำว่าเคยแสดงไปแล้วหรือยัง)

tutorial.js: เป็นระบบ Slide นำเสนอวิธีการเล่นเกม (มีวิดีโอประกอบ) โดยโค้ดจะจับ Event การคลิกซ้าย/ขวา เพื่อเลื่อนสไลด์ เมื่อดูจบสไลด์สุดท้าย โค้ดจะอัปเดตสถานะ hasSeenTutorial = true ลงในข้อมูลผู้ใช้เพื่อไม่ให้แสดงอีก

4. ระบบเกม Box Game (ต่อบล็อกโค้ด)
ไฟล์ที่เกี่ยวข้อง: BoxGame.html, js/boxgame.js, css/boxgame.css

หน้าที่และการทำงานของโค้ด (boxgame.js):
ไฟล์นี้มีการยุบรวมข้อมูลโจทย์ (Data) และตรรกะของเกม (Logic) ไว้ด้วยกันเพื่อให้ดึงข้อมูลได้ทันที

ระบบโจทย์และความยาก: แบ่งโจทย์เป็น 4 ระดับ (Easy, Medium, Hard, Expert)

ระบบ Persistence (จำสถานะการเล่น): โค้ดจะบันทึกสถานะการเล่นลง localStorage ตลอดเวลา (ว่าเล่นระดับไหนอยู่ ข้อที่เท่าไหร่) ทำให้เมื่อกด F5 หรือรีเฟรชหน้าเว็บ โจทย์ข้อเดิมที่ค้างอยู่จะไม่หายไป

ระบบสุ่มโจทย์: ฟังก์ชัน getAvailableQuestions() จะเช็คประวัติว่าด่านนี้ผู้เล่นเคยสุ่มได้ข้อไหนไปแล้วบ้าง เพื่อดึงเฉพาะข้อที่ยังไม่เคยเล่นมาแสดง

การคลิกตอบ: เมื่อผู้เล่นคลิกบล็อกตัวเลือก โค้ดจะนำข้อความนั้นไปใส่ใน Array answerList และวาด (Render) บล็อกใหม่บนกระดานคำตอบ พร้อมกับทำไฮไลต์สีเทาให้กับตัวเลือกด้านล่างเพื่อบอกว่ากดไปแล้ว

การส่งคำตอบ (Submit): เมื่อกดส่ง โค้ดจะเทียบ Array คำตอบของผู้เล่นกับ Array เฉลย

ตอบถูก: แสดงผลสีเขียว เพิ่มคะแนน (SCORE_MAP), เพิ่มสถิติ (Total, Streak) และเช็คเงื่อนไขเพื่อแจก Achievement รวมถึงส่งข้อมูลไปอัปเดตระบบภารกิจรายวัน (Quest)

ตอบผิด: แสดงผลสีแดง เล่นแอนิเมชันสั่น และรีเซ็ตคอมโบ (Streak) ให้เป็นศูนย์

ระบบข้ามด่าน (Skip): หักเหรียญผู้เล่นตามระดับความยาก (Easy 10, Medium 20...) หากเหรียญพอจะข้ามโจทย์ข้อนั้นให้ทันที

5. ระบบเกม Find Game (พิมพ์โค้ด)
ไฟล์ที่เกี่ยวข้อง: duo.html, js/duo.js, css/duo.css

หน้าที่และการทำงานของโค้ด (duo.js):
โครงสร้างการทำงานคล้ายกับ Box Game (มีระบบจำสถานะ, หักเหรียญเพื่อข้ามด่าน, อัปเดตภารกิจ) แต่ต่างกันที่รูปแบบการเล่น:

ผู้เล่นจะต้องพิมพ์โค้ด HTML ลงในช่อง Input ให้ถูกต้องตามคำใบ้

การตรวจคำตอบ (checkAnswer) โค้ดจะนำสตริงที่ผู้เล่นพิมพ์มาลบช่องว่างทิ้ง (replace(/\s+/g, '')) และแปลงเป็นพิมพ์เล็กทั้งหมด (toLowerCase()) จากนั้นนำไปเทียบกับเฉลยเพื่อความยืดหยุ่นในการพิมพ์

เมื่อตอบถูก จะนับจำนวนคำในเฉลยเพื่อนำไปอัปเดตภารกิจ "พิมพ์โค้ดให้ถูกต้อง 3 คำ" เป็นต้น

6. ระบบภารกิจรายวัน (Daily Quests)
ไฟล์ที่เกี่ยวข้อง: quest.html, js/quest.js, css/quest.css

หน้าที่และการทำงานของโค้ด (quest.js):

ระบบเวลานับถอยหลัง: ฟังก์ชัน updateCountdown() จะคำนวณเวลาปัจจุบันเปรียบเทียบกับเวลา 00:00:00 น. ของวันถัดไป แล้วอัปเดตแสดงผลบนหน้าจอแบบ Real-time ทุกๆ 1 วินาที

การแสดงผลภารกิจ: ดึงข้อมูล questProgress จากผู้ใช้มาคำนวณเปอร์เซ็นต์ของหลอด Progress Bar (เช่น ตอบถูก 5 ข้อ, ทำคอมโบ 3 ข้อ, สะสมคะแนน 30 Pts)

การรับรางวัล (Claim Reward): หากทำภารกิจครบตามจำนวน ปุ่มจะเปลี่ยนเป็นสีเขียว เมื่อกดรับรางวัล ฟังก์ชัน claimReward จะบวกเหรียญเข้าบัญชีผู้ใช้ อัปเดตสถานะเป็น "รับแล้ว" และบันทึกลง localStorage

7. ระบบหอเกียรติยศ (Achievements)
ไฟล์ที่เกี่ยวข้อง: achievement.html, js/achievement.js, css/achievement.css

หน้าที่และการทำงานของโค้ด (achievement.js):

ทำหน้าที่อ่านข้อมูล unlockedAchievements ของผู้เล่น

จากนั้นจะลูปหาการ์ด Achievement ในหน้า HTML (อิงจาก ID เช่น box-first, duo-speed)

หากผู้เล่นปลดล็อกแล้ว โค้ดจะปรับแต่ง CSS ผ่าน JavaScript โดยปรับให้การ์ดมีสีสัน (ยกเลิก Grayscale), เปลี่ยนขอบเป็นสีเขียว, และเปลี่ยนไอคอนแม่กุญแจเป็นไอคอนปลดล็อก

8. ร้านค้าธีม (Theme Shop)
ไฟล์ที่เกี่ยวข้อง: themeShop.html, js/themeShop.js, css/themeShop.css

หน้าที่และการทำงานของโค้ด (themeShop.js):

การเรนเดอร์ร้านค้า: ดึงรายการธีม (Light, Dark, Winter, Easter, Halloween) มาแสดง โค้ดจะเช็คว่าผู้ใช้ปลดล็อก (unlockedThemes) ธีมไหนแล้วบ้าง เพื่อเปลี่ยนปุ่มจาก "ซื้อ (Coins)" เป็น "เลือกใช้งาน"

ระบบการซื้อ (buyTheme): เช็คว่าเหรียญของผู้เล่นพอหรือไม่ หากพอจะทำการหักเหรียญ เพิ่มธีมเข้าไปใน Array unlockedThemes บันทึกลงฐานข้อมูล และโชว์ Alert ยืนยัน

ระบบสวมใส่ (equipTheme): เปลี่ยนแปลง Class ของ body ทันที และเก็บชื่อธีมลง localStorage ในคีย์ theme เพื่อให้หน้าเว็บอื่นๆ รู้ว่าตอนนี้ใช้ธีมอะไรอยู่

9. หน้าจัดอันดับ (Leaderboard)
ไฟล์ที่เกี่ยวข้อง: leader_board.html, js/leaderboard.js, css/leaderboard.css

หน้าที่และการทำงานของโค้ด (leaderboard.js):

ดึงข้อมูลผู้เล่นทุกคนในระบบออกมาคำนวณคะแนน โดยเช็คพารามิเตอร์ของ URL (?type=...) ว่าผู้ใช้ต้องการดูบอร์ดรวม, บอร์ด Box Game, หรือบอร์ด Find Game

ใช้คำสั่ง sort เพื่อเรียงลำดับผู้เล่นจากคะแนนมากไปน้อย

ฟังก์ชัน showCurrentUserRank(): ค้นหาว่าตัวผู้เล่นปัจจุบันอยู่ลำดับที่เท่าไหร่ หากลำดับของผู้เล่นอยู่ลึกกว่าข้อมูลที่แสดงในตาราง (ตารางโชว์แค่ 10 คนแรก) โค้ดจะสร้างแถบพิเศษลอยอยู่ด้านล่างสุดของจอ (Sticky bottom) เพื่อให้ผู้เล่นเห็นอันดับของตัวเองเสมอ

10. ระบบโปรไฟล์และการตั้งค่า (Profile & Settings)
ไฟล์ที่เกี่ยวข้อง: profile.html, setting.html, js/profile.js, js/settings.js

หน้าที่และการทำงานของโค้ด:

profile.js: แสดงสถิติรวม ดึงข้อมูลฉายาที่ได้จาก Achievement มาแสดงในช่อง Dropdown ให้ผู้ใช้เลือกตั้งเป็นฉายาตัวเอง (currentTitle)

settings.js: หน้าสำหรับแก้ไขข้อมูลบัญชี

ระบบอัปโหลดรูปโปรไฟล์: เมื่อผู้ใช้อัปโหลดรูป โค้ดจะเรียกใช้ Library ภายนอกที่ชื่อว่า Cropper.js มาสร้างหน้าต่างให้ผู้ใช้สามารถลากและครอปรูปให้เป็นสี่เหลี่ยมจัตุรัสตามที่ต้องการ จากนั้นแปลงรูปเป็นข้อมูลแบบ Base64 string เพื่อเซฟลง localStorage

การเปลี่ยนชื่อและรหัสผ่าน: ตรวจสอบว่า Username ใหม่ซ้ำกับคนอื่นในระบบหรือไม่, ตรวจสอบรหัสผ่านเก่าว่าถูกต้องหรือไม่ก่อนอนุญาตให้ตั้งรหัสผ่านใหม่ เมื่อบันทึกสำเร็จจะอัปเดตข้อมูลและเปลี่ยนชื่อ Session การล็อกอินปัจจุบันด้วย (เพื่อไม่ให้โดนเด้งออกจากระบบ)
